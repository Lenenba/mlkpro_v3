# Malikia Pro Website Copy - Phased User Story

Dernière mise à jour: 2026-04-09

## Goal
Faire évoluer tout le copywriting public de Malikia Pro de façon chronologique, page par page, sans toucher a la structure, aux sections, au layout, aux images, ni a l'architecture des pages.

Le but n'est pas seulement de "mieux écrire".

Le but est de produire un copywriting:
- plus crédible
- plus clair
- plus convaincant
- plus cohérent d'une page a l'autre
- plus compétitif face aux meilleurs acteurs du domaine
- propre en grammaire et orthographe dans chaque langue active

## Why a phased approach
Le document [MALIKIA_PRO_WEBSITE_COPY_REWRITE.md](C:/Users/JulesRogerSombangnen/Herd/mlkpro_v3/docs/MALIKIA_PRO_WEBSITE_COPY_REWRITE.md) reste utile comme brouillon stratégique, mais il ne doit pas être considéré comme la version finale a publier.

Les raisons:
- plusieurs sections sont encore formulées comme des consignes editoriales plutot que comme du vrai copy final
- certaines promesses ont besoin d'être precisées pour mieux rivaliser avec les concurrents
- chaque langue doit être revue comme une version native, pas comme une simple traduction
- il faut valider la qualité page par page pour garder un niveau premium constant

## Non-goals
- changer la structure des pages
- ajouter de nouvelles sections sans blocage majeur
- changer le layout
- changer les images
- faire une traduction littérale mot a mot
- publier du copy non valide linguistiquement

## Editorial quality bar
Chaque page réécrite doit respecter ce niveau de qualité:
- grammaire propre
- orthographe propre
- ton SaaS premium et crédible
- promesse orientee vers le resultat business, pas simple liste de fonctions
- clarté immediate en moins de quelques secondes de lecture
- CTA plus nets et plus actionnables
- cohérence avec ce que Malikia Pro offre réellement
- aucune phrase vide, générique, ou interchangeable avec n'importe quel SaaS

## Competitiveness standard
L'objectif n'est pas de copier Jobber, Housecall Pro, Workiz ou ServiceTitan.

L'objectif est d'atteindre un niveau comparable sur:
- la clarté de la promesse
- la confiance degagee par le message
- la précision des bénéfices
- la fluidite entre sections
- la sensation de produit sérieux, moderne, simple, et robuste

Le niveau attendu en fin de chantier:
- une homepage qui inspire confiance très vite
- des pages modules qui expliquent une vraie valeur operationnelle
- des pages solutions qui racontent un workflow complet
- des pages industries qui parlent comme si le produit comprenait réellement le métier

## Working method for every phase
Chaque phase doit suivre exactement le même workflow:

1. Audit de la page actuelle
- comprendre l'intention actuelle
- identifier les zones faibles
- verifier si le niveau concurrentiel est suffisant

2. Ecriture de la version source
- réécrire chaque section existante sans changer la structure
- produire un copy plus clair, plus premium, plus persuasif
- verifier l'alignement produit

3. Adaptation FR
- produire une vraie version française naturelle
- corriger grammaire, orthographe, rythme, et ton
- eviter les formulations trop traduites ou trop lourdes

4. Adaptation EN
- produire une vraie version anglaise naturelle
- garder un ton SaaS fort et moderne
- eviter les formulations génériques ou scolaires

5. QA editoriale
- cohérence entre sections
- cohérence avec les autres pages
- cohesion du vocabulaire de marque
- qualité des CTA
- absence de promesses trompeuses

6. Validation
- validation page
- puis seulement passage a la phase suivante

## Deliverables required for each page
Chaque phase doit livrer:
- la page cible
- la liste des sections existantes conservees
- une version source approuvee
- une version FR finale
- une version EN finale
- les CTA finaux si la page en a
- les points de vigilance copy/positionnement

## Definition of done for each page
Une page est considérée terminee seulement si:
- le copy est final dans toutes les sections existantes
- la promesse est claire
- la page semble competitive face a des acteurs SaaS sérieux
- le texte FR est publiable
- le texte EN est publiable
- il n'y a plus de placeholder ni de note interne
- les CTA sont cohérents avec le parcours utilisateur
